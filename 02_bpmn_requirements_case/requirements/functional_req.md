# Функциональные требования к системе документооборота (Битрикс24)

## Проект: строительство метро «Юго-Западная», подрядчик ООО «Розенберг»

### 1. Управление документами (акты КС-2, КС-3)
- 1.1. Прораб должен иметь возможность загрузить файл акта (PDF, до 20 МБ) и заполнить карточку с полями:
  - Номер акта
  - Дата составления
  - Отчётный период (месяц, год)
  - Объект (фиксированный справочник: «Юго-Западная, этап 2»)
  - Тип работ (из справочника)
  - Сумма (рубли)
  - Субподрядчик (выбор из списка, если есть)
  - Флаг «Требуется Росстройконтроль» (да/нет)
- 1.2. После сохранения документ автоматически переходит в статус «Черновик».
- 1.3. При отправке на проверку статус меняется на «На проверке в ПТО», запускается SLA-таймер (1 рабочий день).

### 2. Маршрутизация и согласование
- 2.1. ПТО получает задачу проверить документ с кнопками: «Утвердить» или «Отклонить с комментарием».
- 2.2. При утверждении ПТО статус меняется на «Согласовано ПТО», документ автоматически отправляется генподрядчику (через API внешней системы или по email со ссылкой).
- 2.3. При отклонении статус меняется на «Доработка», прорабу приходит уведомление (Telegram + email). После исправления прораб отправляет документ повторно, и цикл возвращается к п. 2.1.
- 2.4. Генподрядчик получает уведомление со ссылкой на гостевой доступ. Может подписать ЭЦП или вернуть с замечаниями.
- 2.5. Если генподрядчик не отвечает в течение 3 рабочих дней — автоматическая эскалация руководителю генподрядчика.

### 3. Интеграции
- 3.1. REST API: при отправке генподрядчику система должна вызывать внешний API `POST /api/v1/act/approve` с телом:
  ```json
  {
    "act_number": "КС-2-123",
    "date": "2025-10-01",
    "amount": 1250000,
    "file_url": "https://bitrix24.ru/...",
    "contractor": "Розенберг"
  }
- 3.2. Веб-хук от генподрядчика: при получении статуса «подписан» Битрикс24 обновляет статус документа и уведомляет ПТО и прораба.
- 3.3. Для Росстройконтроля: при флаге «Требуется Росстройконтроль» после подписания генподрядчиком система автоматически отправляет документ в Единую систему Росстройконтроля через их API (формат XML, по спецификации).
